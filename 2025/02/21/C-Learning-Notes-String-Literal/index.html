C Learning Notes - String Literal

I am exploring the possibility of entering the embedded systems development field, so I decided to practice my C programming skills. My knowledge of C is fairly limited, as I only learned the basics during my undergraduate studies a few years ago. Because of that, I’m not ashamed to appear like a beginner while reviewing it. Today, I attempted to solve a problem from LeetCode using C.

The problem: https://leetcode.com/problems/longest-common-prefix/description/.

For now, my view on how to solve a problem like LeetCode is: first, you come up with a logical sequence of steps; second, try to translate your logic into the programming language in your mind roughly; third, code while considering potential bugs in your code. The key point is—coding should be the last step.

Copilot summarized my logic very clearly (I relied heavily on Copilot this time since I found myself forgetting most of the syntax of the language):

  1. Find the shortest word in the strs array.
  2. Use the shortest word as a reference to compare characters with other words literally.
  3. If a mismatch is found, return the common prefix up to that point.
  4. If no mismatch is found, return the entire shortest word.

The shortest word is required here to avoid the index being out of bounds.

According to Copilot, “a segmentation fault (often abbreviated as segfault) is a specific kind of error caused by accessing memory that does not belong to you. It’s a flaw in the program that occurs when the program tries to read or write to an illegal memory location.”
And, “shortestWord is a string literal, modifying it with shortestWord[i] = '\0'; will cause undefined behavior and likely a segmentation fault.”
